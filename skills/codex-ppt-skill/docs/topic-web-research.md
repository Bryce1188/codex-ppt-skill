# 主题/画展/活动联网调研

当用户只给一句话主题，而不是本地项目目录时，使用这个流程。

## 适用场景

- “生成某某画展 PPT”
- “帮我做一个某艺术家的展览介绍”
- “做一个某城市/景点/博物馆的分享 PPT”
- “生成某品牌、某发布会、某公司、某人物的介绍 PPT”
- “做一个某技术趋势/社会热点/新闻事件的汇报 PPT”

## 先判断输入类型

把任务分成三类：

1. **项目型**：有本地代码、README、运行截图、项目文件夹。走 `project-report-audit.md`。
2. **材料型**：用户给了文章、PDF、报告、笔记、大纲。先读材料，必要时联网补背景。
3. **主题型**：用户只给一句话或一个名称。必须联网调研后再做 PPT。

不要把主题型任务误判成项目型。比如“生成某某画展 PPT”通常是主题型，不需要扫描代码项目。

## 联网搜索顺序

优先级从高到低：

1. 官方网站：展览官网、博物馆/画廊/美术馆官网、主办方页面、艺术家官网、品牌官网。
2. 官方发布：新闻稿、活动公告、票务页面、官方社媒公告。
3. 权威媒体：主流媒体、艺术媒体、行业媒体、出版社、学术机构。
4. 资料库：百科、博物馆数据库、艺术家档案、公开论文或报告。
5. 图片来源：官方海报、展览现场图、作品图、场馆图；记录来源，避免伪造素材出处。

## 必须核验的信息

画展/展览类至少核验：

- 展览名称
- 时间
- 地点
- 主办方/策展方/场馆
- 艺术家或主题背景
- 核心展品、展区或看点
- 票务/开放时间，只有用户需要时才放进 PPT
- 图片来源和版权风险

人物/品牌/公司类至少核验：

- 名称和身份
- 关键时间线
- 代表作品/产品/事件
- 近期动态
- 影响、争议或限制，按任务需要取舍

趋势/新闻类至少核验：

- 事件发生时间
- 关键数据和发布机构
- 多方观点
- 最新进展
- 不确定信息

## 资料卡模板

```text
主题：
任务类型：
受众：
建议页数：

确认事实：
- 事实 | 来源 | 日期

关键看点：
- 看点 | 证据/来源 | 可视化想法

可用图片/素材：
- 路径或 URL | 内容 | 来源 | 是否适合入 PPT

不确定或缺失：
- 信息 | 为什么不确定 | 是否需要问用户
```

## PPT 故事线建议

画展/艺术展默认结构：

1. 封面：展览名称和最强视觉。
2. 展览一句话：这场展览为什么值得看。
3. 背景：艺术家、主题、时代或策展背景。
4. 核心看点：展览中最重要的 3-4 个亮点。
5. 代表作品/展区：用图片和简短解读串联。
6. 观展路线或体验：如果有场馆/展区信息。
7. 视觉风格与情绪：适合做强视觉页面。
8. 总结：展览价值、适合人群、推荐理由。

公开主题/人物/品牌默认结构：

1. 封面：主题和主视觉。
2. 一句话结论：这个主题为什么重要。
3. 背景和时间线。
4. 核心内容或关键事件。
5. 证据、案例或代表作品。
6. 影响和价值。
7. 争议、限制或风险。
8. 总结和下一步。

## 引用和诚实性

- 不要编造展览、时间、地点、展品、票价、数据或评价。
- 只找到单一来源时，避免写得过于绝对。
- 资料冲突时，优先官方来源，并在内部 notes 中记录冲突。
- 最终回复要列出主要来源；PPT 中可在页脚或附录放简短来源。
- 对图片版权不确定时，说明“需用户确认授权/可用性”。
